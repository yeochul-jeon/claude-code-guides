## [Phase 2] 구조 개선 — 디렉토리 분리 + 교차 참조

- **결정**: 범용 가이드와 CJ 전용 가이드를 디렉토리로 분리하기로 결정
  - `guides/` — 범용 Claude Code 가이드 4개
  - `internal/` — CJ 전용 문서 (Confluence CLI)
- **시도**: 
  1. `mkdir -p guides internal`로 디렉토리 생성
  2. `git mv`로 파일 이동 (git 이력 보존)
  3. README.md, CLAUDE.md 내 경로를 새 구조에 맞게 업데이트
  4. 각 가이드 문서 상단에 "관련 문서" 교차 참조 섹션 추가
  5. 모든 문서의 변경일을 2026-04-01로 갱신
- **오류**: 
  - `git mv`로 이동한 파일을 새 경로로 Read하지 않고 Edit 시도 → `File has not been read yet` 에러 4건 발생
- **해결**: 
  - 이동된 파일을 새 경로(`guides/...`)로 다시 Read한 후 Edit 재실행하여 정상 적용
- **메모**:
  - 교차 참조는 상대 경로 사용 (같은 `guides/` 내이므로 파일명만으로 링크)
  - Confluence CLI 가이드는 교차 참조 대상에서 제외 (성격이 다름)
  - `git mv` 사용으로 rename 이력이 보존됨
